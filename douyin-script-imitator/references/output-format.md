# 输出格式规范

## 第1段：原视频解析

```
🎬 原视频解析

📹 形式：[工厂探店/大字报/危机方案/...]
📌 标题：[推断标题，第一个！或。之前的短句]
📝 描述：[推断描述，标题之后、标签之前的剩余内容]
🏷️ 标签：[#标签1 #标签2 ...]
👤 账号：昵称 | 粉丝 | 类型
📊 数据：点赞/评论/收藏/分享
🔗 链接：[URL]
```

## 第2段：原视频文案

```
📝 原视频文案：
[转写文案，不使用引用格式，直接输出文本]
```

## 第3段：结构拆解 + 企业摘要

```
🔍 结构拆解：
| # | 原文 | 功能 | 技巧 |
|---|------|------|------|

🏢 仿写的企业信息摘要：[≤200字]
```

## 第4段起：仿写脚本（每条脚本单独一段消息）

```
✍️ 仿写脚本 X ｜ [角度]

🎬 画面脚本：(大字报的不用输出)
| 时段 | 画面 | 景别 | 文案 |
|------|------|------|------|

📝 文案：
[完整文案]
博眼球或噱头文案(≤15字,大字报才输出)
🎤 描述语：[≤50字，用于发布时的作品描述]
📌 标题：[≤15字，用于发布时的视频标题,大字报不需要]
🏷️ 标签：[#标签1 #标签2 ...]
```

---

## 字段说明

| 字段 | 对应抖音发布 | 说明 |
|------|-------------|------|
| 标题 | 发布时的标题输入框 | 视频封面/列表页显示的短标题，≤15字 |
| 描述语 | 发布时的作品描述输入框 | 视频详情页展示的描述文案，≤50字 |
| 标签 | 话题标签 | 发布时选择的#话题，5-8个 |
| 文案 | 口播内容 | 拍摄时念的文案，与描述语可相同可不同 |
