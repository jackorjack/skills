### 读取企业资料（记忆优先 → 文件兜底 → 同步兜底）

**核心原则：优先从 memory 检索，避免重复加载大文件，命中即用。**

执行顺序：
1. **memory_search 检索**：用以下 query 组合并发检索企业信息（`corpus=memory`），每轮至少 3 条 query：
   - `"企业名称 品牌定位 核心业务 产品线"` → 获取企业基础信息和产品线
   - `"技术壁垒 核心性能 适用场景 用户痛点"` → 获取产品卖点和行业信息
   - `"客户案例 服务数据 行业影响力 竞争优势"` → 获取信任背书和案例
   - 若企业有明确行业关键词（如"鞋垫""机械设备"），追加 `"{行业关键词} 产品 工艺 应用场景"`
2. **memory_get 补全**：检索命中的片段若被截断（含"…"），用 `memory_get` 按路径+行号拉取完整内容
3. **文件兜底**：若 memory 未命中或内容明显不足（少于 3 条有效信息片段），降级读取 `workspace/wiki/` 目录下 md 文档
4. **同步兜底**：若 workspace/wiki/ 目录下无文档，执行技能 `enterprise-kb-sync` 将企业知识库同步到本地，同步完成后回到步骤 1 重新走记忆检索

**注意**：
- memory_search 返回的 `snippet` 可能不完整，关键信息必须用 memory_get 补全
- 不要仅凭 memory_search 的 snippet 判断信息充足，需确保读取到完整段落
- 企业 wiki 文档已自动索引到 memory，优先走记忆通道

### 企业及产品梳理（自动输出）
生成三部分：

**1. 用户分析**（按产品线/行业拆分）

每个行业输出表格：应用场景 | 用户痛点 | 业务优势 | 用户爽点
- 场景要具体（谁+什么情况+需要什么）
- 痛点要量化（数字+对比+后果）
- 优势要差异化（对应痛点的独特方案）
- 爽点要可感知（省了/赚了/解决了什么）
- 每个行业3-5个场景，自动扩展补充

产品分类术语自动选择：制造业→产品系列/产品线，电商→商品类目，服务→服务类型/解决方案

**2. 产品卖点梳理**
每类产品：应用场景 | 核心卖点 | 差异化卖点 | 口语化传播话术

**3. 企业优势梳理**
企业实力 | 信任背书 | 可包装方向 | 其他优势（可搜索补充）

### 读模板
用户选择后，**只加载对应文件**。

**模板目录选择逻辑（行业优先 → public 兜底）**：

1. 从企业行业/产业信息中提取关键词（如「制造业 — 鞋材 / 功能鞋垫 / 足部护理」→ 关键词：`鞋材` `功能鞋垫` `足部护理` `制造业`）
2. 遍历 `references/` 下的子目录，用关键词与目录名做模糊匹配，找到匹配度最高的行业目录（如 `references/鞋垫/`）
3. 若行业目录下存在对应的 `{视频形式}-{类型}/` 子目录且包含目标模板文件，则**从行业目录加载**
4. 若行业目录不存在或无对应模板文件，**降级到 `references/public/` 加载**

**路径规则**：
- 优先路径：`references/{行业目录}/{视频形式}-{类型}/{选题名称}.md`
- 降级路径：`references/public/{视频形式}-{类型}/{选题名称}.md`

**匹配逻辑**：模板文件名 = 选题名称（如选题「老板人设」→ `老板人设.md`，选题「痛点爽点」→ `痛点爽点.md`），无需额外映射。


### 生成完整方案

#### 大字报模式

**核心创作公式（五要素法）**：
```
应用场景 → 用户痛点 → 用户爽点 → 独特优势 → 公司实力
```

**公式详解**：
| 要素 | 说明 | 写作要点 |
|---|---|---|
| ① 应用场景 | 行业客户最关心的问题点，具体使用情境（谁+什么情况+需要什么） | 一句话带出，让客户觉得"说的就是我" |
| ② 用户痛点 | 客户最头疼的问题，行业普遍存在的困扰 | 必须量化！用数字+对比+后果，戳中痛处 |
| ③ 用户爽点 | 客户最渴望得到的好处，解决问题后的理想状态 | 可感知！"省了多少/赚了多少/解决了什么" |
| ④ 独特优势 | 你能解决痛点的差异化方案，别人做不到的或者做得更好的 | 对应痛点，精准打击，突出"只有我能" |
| ⑤ 公司实力 | 用行业top1/领先的数据证明你的能力和可信度 | 用具体数据（16年/100+专利/月产200万双）代替形容词 |

**文案结构（5行法）**：

文案直接作为画面叠加文字，**无独立标题**，共5行，**总字数≤100字**：

| 行号 | 内容定位 | 写作要点 |
|---|---|---|
| 第1行 | **开头含噱头** | 噱头博眼球，吸引用户停留 |
| 第2-3行 | **中间讲业务卖点** | 圈人群、应用场景、用户痛点、用户爽点 |
| 第4-5行 | **结尾点明业务** | 点名业务范围，亮企业实力，或引导互动留资 |

**输出结构**：
1. **视频文案（5行大字叠加）**：≤100字，严格按5行结构排列，直接作为视频画面主体文字
2. **话题标签**：3-5个精准标签，覆盖产品词+行业词+人群词
3. **封面方案×2**：每个含封面文案+设计建议+图片提示词（中文）

#### 真人出镜/混剪模式
1. **视频文案（分段+纯文案）**：15-45秒，抖音快节奏，开场钩子+痛点共鸣+方案输出+行动号召。提供分段文案（带时长）+ 纯文案
2. **9个视频标题（表格）**：揭秘式 | 利益式 | 悬念式 | 数字式 | 对比式 | 好奇式 | 专业式 | 故事式 | 痛点式
3. **分镜脚本（表格）**：景别 | 画面内容 | 台词/字幕 | 时长 | BGM/音效 | 拍摄要点
4. **封面方案×2**：每个含封面文案+设计建议+图片提示词（中文）

## 核心原则
1. **抖音适配**：15-45秒，开场3秒必须有钩子
2. **参考风格不照抄**：学习模板的节奏感、口语化、冲突感、结构融入企业实际信息
3. **大字报五要素公式**（必须遵守）：`应用场景 → 用户痛点 → 用户爽点 → 独特优势 → 公司实力`
   - 无独立标题，文案即画面主体，5行结构，≤100字
   - 第1行（开头含噱头）：场景+痛点+爽点融合，制造钩子
   - 第2-3行（中间讲业务卖点）：圈人群+圈行业+应用场景+痛点/爽点对比+独特优势
   - 第4-5行（结尾点明业务）：点名业务范围，亮企业实力，引导互动留资
   - 痛点必须量化（数字+对比+后果），爽点必须可感知，实力用数据证明
